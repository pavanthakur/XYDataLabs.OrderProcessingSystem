```yaml
name: Deploy API to Azure App Service

on:
  push:
    branches:
      - main
    paths:
      - 'XYDataLabs.OrderProcessingSystem.API/**'
      - 'XYDataLabs.OrderProcessingSystem.Application/**'
      - 'XYDataLabs.OrderProcessingSystem.Domain/**'
      - 'XYDataLabs.OrderProcessingSystem.Infrastructure/**'
      - 'XYDataLabs.OrderProcessingSystem.Utilities/**'
      - 'XYDataLabs.OpenPayAdapter/**'
  workflow_dispatch:

env:
  DOTNET_VERSION: '8.x'
  AZURE_WEBAPP_NAME: 'orderprocessing-api-yourname'  # CHANGE THIS
  AZURE_WEBAPP_PACKAGE_PATH: './publish'
  PROJECT_PATH: 'XYDataLabs.OrderProcessingSystem.API/XYDataLabs.OrderProcessingSystem.API.csproj'

jobs:
  build:
    runs-on: windows-latest
    permissions:
      contents: read

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: ${{ env.DOTNET_VERSION }}

      - name: Restore dependencies
        run: dotnet restore ${{ env.PROJECT_PATH }}

      - name: Build
        run: dotnet build ${{ env.PROJECT_PATH }} --configuration Release --no-restore

      - name: Publish
        run: dotnet publish ${{ env.PROJECT_PATH }} --configuration Release --no-build --output ${{ env.AZURE_WEBAPP_PACKAGE_PATH }}

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: dotnet-app
          path: ${{ env.AZURE_WEBAPP_PACKAGE_PATH }}

  deploy:
    runs-on: windows-latest
    needs: build
    permissions:
      id-token: write
      contents: read
    environment:
      name: 'Production'
      url: ${{ steps.deploy-to-webapp.outputs.webapp-url }}

    steps:
      - name: Download artifact
        uses: actions/download-artifact@v4
        with:
          name: dotnet-app

      - name: Login to Azure
        uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZUREAPPSERVICE_CLIENTID }}
          tenant-id: ${{ secrets.AZUREAPPSERVICE_TENANTID }}
          subscription-id: ${{ secrets.AZUREAPPSERVICE_SUBSCRIPTIONID }}

      - name: Deploy to Azure Web App
        id: deploy-to-webapp
        uses: azure/webapps-deploy@v3
        with:
          app-name: ${{ env.AZURE_WEBAPP_NAME }}
          package: .
```

## **üêõ Common Issues and Solutions**

### **Issue 1: GitHub Actions Workflow Fails**

**Symptom:** Workflow shows red X, deployment fails

**Common Causes & Solutions:**

1. **Build Fails:**
   ```
   Error: Project not found at path
   ```
   **Solution:** Verify `PROJECT_PATH` in workflow matches actual project location

2. **Authentication Fails:**
   ```
   Error: Login failed
   ```
   **Solution:**
   - Check GitHub secrets are set correctly
   - Re-authorize GitHub in Azure Deployment Center
   - Verify client-id, tenant-id, subscription-id match

3. **Publish Fails:**
   ```
   Error: NuGet package restore failed
   ```
   **Solution:**
   - Ensure all NuGet packages are available
   - Check network connectivity during build
   - Add `--no-cache` flag to dotnet restore

**Debugging Steps:**
1. Click on failed workflow in GitHub Actions
2. Expand failed step
3. Read error message carefully
4. Check workflow file for typos
5. Verify all paths and names match your setup

---

### **Issue 2: Application Not Accessible (404/500 Errors)**

**Symptom:** Opening Web App URL shows error page

**404 Not Found:**
- **Cause:** Application not deployed or wrong URL
- **Solution:**
  - Verify deployment completed successfully
  - Check URL spelling
  - Ensure application published to wwwroot for UI

**500 Internal Server Error:**
- **Cause:** Application crash or configuration error
- **Solution:**
  - Check Application Insights logs
  - View Log Stream in Azure Portal
  - Verify connection strings and settings
  - Enable detailed errors in staging

**Debugging Steps:**
1. Azure Portal ‚Üí Your Web App ‚Üí Log stream
2. Look for startup errors
3. Check Application settings are correct
4. Verify runtime stack matches (.NET 8)
5. Review deployment logs in Deployment Center

---

### **Issue 3: UI Cannot Connect to API**

**Symptom:** UI loads but shows errors when accessing data

**CORS Errors:**
```
Access to fetch blocked by CORS policy
```
**Solution:**
- Add CORS configuration in API `Program.cs`:
```csharp
builder.Services.AddCors(options =>
{
    options.AddDefaultPolicy(policy =>
    {
        policy.WithOrigins("https://orderprocessing-ui-yourname.azurewebsites.net")
              .AllowAnyHeader()
              .AllowAnyMethod();
    });
});

// After app is built:
app.UseCors();
```

**Wrong API URL:**
- **Solution:** Verify `ApiSettings__BaseUrl` in UI settings points to correct API URL

**Network/DNS Issues:**
- **Solution:** Test API directly in browser first, then test UI
